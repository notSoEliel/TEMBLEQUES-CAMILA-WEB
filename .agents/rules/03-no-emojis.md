# No Emojis in Development and Documentation

To maintain a professional, clean, and high-quality standard for the project, the use of emojis is strictly prohibited in the following areas:

## Scope
- **Source Code**: No emojis in comments or string literals unless explicitly required by business logic.
- **Commit Messages**: All commit messages must be text-only and professional.
- **Documentation**: No emojis in `README.md`, `PRD.md`, or any other documentation files.
- **UI Components**: Emojis should not be used as icons; use professional icon libraries like `lucide-react` instead.

## Rationale
Emojis often detract from the premium and "production-grade" aesthetic the project aims to achieve through its Glassmorphism design and Antigravity standards. Professional documentation and code should rely on clear language and meaningful icons rather than informal symbols.
