# Reglas de Documentación - Tembleques Camila

1. **Documentación del Proyecto**:
   - Cada funcionalidad implementada debe tener o actualizar la documentación en la carpeta `docs/`.
   - Utilizar archivos `.md` separados para mejorar el entendimiento (ej. `architecture.md`, `database.md`).

2. **Uso de Mermaid**:
   - Se debe incluir diagramas de Mermaid en la documentación para representar flujos complejos, diagramas de base de datos o arquitectura de sistema.

3. **Mantenimiento**:
   - Al actualizar una funcionalidad o esquema en la base de datos, asegurarse de actualizar paralelamente su representación en `docs/`.
   - La documentación debe reflejar fielmente la integración de servicios de terceros (Clerk, Stripe).
